# 직장인(職匠人) 국비지원 교육 페이지 - 배포 안내

## 들어있는 파일
- gov_edu.html      : 국비지원 교육 허브 (검색창 + 12개 카테고리)
- gov_courses.html  : 키워드별 결과 페이지 (지역필터·과정목록·신청팝업·추천·정보·하단지역네비)
- netlify.toml      : /api 요청을 프록시 서버로 연결하는 설정 (중요!)

## 배포 방법
1. 기존 사이트 폴더에 위 3개 파일을 넣습니다.
   (이미 netlify.toml이 있다면 [[redirects]] 블록만 합쳐주세요)
2. Netlify에 평소처럼 배포(드래그 업로드 또는 git push)합니다.
3. 배포 후 접속:
   - 허브:   https://(사이트주소)/gov_edu.html
   - 결과:   https://(사이트주소)/gov_courses.html?keyword=요양보호사

## 동작 원리
- 페이지는 같은 도메인의 /api/courses 를 호출합니다.
- netlify.toml이 그 요청을 프록시 서버(http://101.79.21.81:3000)로 대신 연결합니다.
- 그래서 https 차단(혼합 콘텐츠) 없이 작동하고, API 키는 프록시 서버에만 있어 노출되지 않습니다.

## 꼭 해주실 것 2가지
1) 헤더/푸터 교체
   - 두 파일 안의 <!-- 임시 헤더 --> / <!-- 임시 푸터 --> 부분을
     대표님 사이트의 실제 header / footer 코드로 바꿔주세요. (그 외는 건드릴 필요 없음)

2) 프록시 rate limit 상향 (권장)
   - Netlify 프록시를 쓰면 모든 요청이 Netlify 서버 IP로 들어가서,
     기존 "IP당 1분 30회" 제한에 금방 걸릴 수 있습니다.
   - 서버에서 아래 한 줄 실행하면 1분 300회로 올라갑니다:
     cd /var/www/proxy && sed -i 's/max: 30/max: 300/' index.js && pm2 restart proxy

## 키워드별 정보 글 추가
- gov_courses.html 안의 CONTENT 객체에 키워드를 추가하면
  "배우는 내용 / 자격증 / 추천 / 전망"이 그 키워드 맞춤으로 표시됩니다.
- 지금은 "요양보호사"만 채워져 있고, 나머지는 일반 안내문이 자동으로 들어갑니다.
- 원하시면 키워드별 원본 글을 계속 채워드릴 수 있어요.

## 참고
- 지역은 시/도 → 시/군/구까지 필터됩니다. (동/읍/면은 고용24 목록 데이터에 없어 불가)
- 과정은 "오늘 기준 개강 임박순"으로 정렬되며, 페이지당 5개씩 보여줍니다.
- 데이터가 없는 지역은 "해당 지역에는 교육이 없어요"로 안내됩니다.
- 모든 교육 정보는 고용24(work24.go.kr) 공공데이터를 실시간으로 불러옵니다.
